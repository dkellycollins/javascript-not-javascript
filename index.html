Rules of the Game

Javascript

Template Literals

  • Introduced in ES6
  • Basically a short-hand syntax for building string values.
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Template_literals
const num = 1000;
const obj = {};
const arr = [];

Javascript

Constant variables

  • Introduced in ES6
  • Prevents the variable from being re-assigned, unlike var or let.
  • Does not prevent a variable from being modified.
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/const
function log(strings) {
  console.log(strings.raw[0]);
}

log`string text line 1 \n string text line 2`;

Javascript

Tagged Template Literals

  • Introduced in ES6
  • Uses a custom function to format the provided string.
  • The returned value does not have to be a string.
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Template_literals
const squared = 2 ** 2; //4
let cubed = 4;
cubed **= 3; //64

Javascript

Exponentiation

  • Introduced in ES6
  • Short-hand for Math.pow
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Arithmetic_Operators
function getDisplay(value) {
  return value ?? 'Unknown';

Not Javascript

Nullish coalescing

  • Similar to "||", but only with null or undefined values
  • Babel
    https://babeljs.io/docs/en/babel-plugin-proposal-nullish-coalescing-operator
  • Stage 3 proposal
    https://github.com/tc39/proposal-nullish-coalescing 
const config = {
  item1: { value: 'A' },
  item2: null
};

const value1 = config.item1?.value;
const value2 = config.item2?.value;

Not Javascript

    https://github.com/TC39/proposal-optional-chaining

Javascript

Get/Set

Javascript

Async/await

  • Introduced in ECMAScript 2017
  • Short-hand for handling Promises.
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/async_function
function* range(start, end) {
  for (let i = start; i < end; i += 1) {
    yield i;
  }
}

for (let i of range(0, 100)) {
  console.log(i);
}

Javascript

Generators

  • Introduced in ES6
  • Can "emit" multiple values
  • How Typescript implemented async/await in ES6.
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Generator
async function* range(start, end) {
  for (let i = start; i < end; i += 1) {
    yield await Promise.resolve(i);
  }
}

async function() {
  for await (let num of range(0, 100)) {
    console.log(num);
  }
}

Async Iteration

  • Iterates through an async iterator, waiting for each value to be emitted
  • Stage 3 proposal
    https://github.com/tc39/proposal-async-iteration
  • Typescript
    https://www.typescriptlang.org/docs/handbook/release-notes/typescript-2-3.html#async-iteration
  • Babel
    https://babeljs.io/docs/en/babel-plugin-proposal-async-generator-functions 
function getLength(vector) {
  case (vector) {
    when { x, y, z } -> Math.sqrt(x ** 2 + y ** 2 + z ** 2)
    when { x, y } -> Math.sqrt(x ** 2 + y ** 2)
    when [...etc] -> vector.length
  }
}

Parameter properties

  • Syntactic sugar for declaring and accepting properties through the class constructor.
  • Can assign any access modifier to enable this.
  • https://www.typescriptlang.org/docs/handbook/classes.html

const input1: Input = '123';

Union Type

  • Allows a variable to be one of many types.
  • https://www.typescriptlang.org/docs/handbook/advanced-types.html#union-types
  • Variant Type in ReasonML:
    https://reasonml.github.io/docs/en/variant 
function isNumber(value: number | undefined | null): value is number {
  return value !== undefined && value !== null;
}

const values: Array<number | undefined | null> = [
  1, undefined, null
];
const validValues: Array<number> = values.filter(isNumber);

Type Guards

  • Custom method that can be used to verify a value is a specific type.
  • https://www.typescriptlang.org/docs/handbook/advanced-types.html
interface Column<T> {
  id: string;
  field: keyof T;
}

interface Data {
  item1: string;
  item2: string;
}

const columns: Array<Column<Data>> = [
  { id: 'col1', field: 'item1' },
  { id: 'col2', field: 'item2' },
  // causes compiler error
  // { id: 'col3', field: 'item3' } 
];
